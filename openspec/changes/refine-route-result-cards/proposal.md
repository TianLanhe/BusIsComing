## Why

目前主頁路線結果卡片已替代表格，但資訊位置與用戶討論後確認的掃讀方式仍不一致：價格仍在第一行競爭視覺焦點，「總耗時」命名過長，等候狀態文案與顏色不足以快速區分有車和無車。現在需要把已確認的 UI 方案固化到 OpenSpec，避免後續實作時因圖片上下文缺失而偏離。

## What Changes

- 在結果清單上方新增整體摘要：左側顯示 `共 X 條路線，Y 條直達`，右側顯示 `更新時間：HH:MM:SS`。
- 路線結果卡片保持簡潔文字路線展示，不使用號牌樣式；單段顯示 `788`，多段使用 `789 → 15`。
- 卡片第一行左側展示路線文字，右側展示等候狀態；價格移至卡片分隔線下方的信息區。
- 有可用 ETA 時等候文案改為 `等候 X 分鐘`，使用主綠色突出；無可用車輛時文案改為 `暫無車輛`，使用灰色，不使用橙色。
- 將所有面向用戶的 `總耗時` 改為 `耗時`，包含卡片字段與排序控件文案。
- 卡片底部信息區展示 `價格 · 耗時 · 步行`，不展示轉乘次數。
- 保留現有排序能力與默認耗時升序排序，只調整排序控件文案與結果摘要展示。

## Capabilities

### New Capabilities

無。

### Modified Capabilities

- `route-query-results-layout`: 調整主頁結果摘要、卡片視覺層級、價格位置、等候狀態位置和狀態顏色。
- `bus-route-result-table`: 調整結果卡片字段文案與排序控件文案，將 `總耗時` 改為 `耗時`。
- `bus-route-walking-distance-table`: 調整價格、耗時、步行距離在補充信息區的展示規則，確保不展示轉乘次數。

## Impact

- 影響主頁結果區 XML 佈局：`activity_main.xml`、`item_bus_route.xml`。
- 影響主頁結果綁定與摘要更新邏輯：`MainActivity.kt`、`BusRouteAdapter.kt`。
- 可能新增或調整顏色資源，用於區分可用等候與 `暫無車輛` 灰色狀態。
- 不改動 Citybus HTTP API、ETA 查詢、路線解析、去重或排序算法。
