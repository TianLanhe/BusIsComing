## Why

当前主界面的结果表格使用固定宽度，未查询时表头与查询后的结果行容易出现宽度不一致；路线选择、地点搜索和路线编辑流程也缺少足够清晰的反馈与快捷操作。优化这些界面细节可以提升移动端可读性，并减少用户在管理常用路线时的重复输入。

## What Changes

- 调整主界面结果表格为屏幕自适应宽度，未查询表头与查询后结果行保持同宽，并让列宽、行高和字号在手机屏幕上保持可读。
- 保持结果表头文案不变，优化结果行布局，使常见屏幕一页大约可显示 5 到 7 行。
- 将主界面路线选择框的折叠态改为主标题/副标题展示：主标题为路线名称，副标题为 `起点 -> 终点`；下拉列表保持当前展示方式。
- 优化“管理路线”入口按钮颜色，使其看起来是可点击按钮且与“查询”主按钮区分。
- 路线管理页新增“克隆”操作，可复制已有路线进入新增编辑页，并预填名称、起点和终点。
- 路线编辑页新增起点/终点交换按钮、页面内返回按钮、整体居中布局和更友好的输入提示文案。
- 保存路线时禁止新增或编辑成与其他路线三要素完全一致的配置；编辑原路线且未改变三要素时允许保存。
- 起点/终点搜索时提供非阻塞加载动画与辅助文案，输入期间可以继续编辑并取消旧搜索反馈。

## Capabilities

### New Capabilities
- `route-query-results-layout`: 定义主界面查询结果表格的响应式宽度、列宽、行高和字号适配要求。
- `route-management-actions`: 定义路线管理入口、克隆路线、重复路线校验、路线编辑页返回与布局等交互要求。

### Modified Capabilities
- `route-place-storage-and-query`: 将主界面已保存路线选择框折叠态从单行标签展示调整为主标题/副标题展示。
- `route-place-selection`: 增加起点/终点搜索加载反馈、起终点交换按钮和更友好的地点输入提示要求。
- `place-search-api`: 增加地点搜索进行中状态，以便界面展示非阻塞加载反馈。

## Impact

- 影响主界面布局与交互：`activity_main.xml`、`MainActivity.kt`、结果表头/列表相关适配器与样式资源。
- 影响路线管理页：`activity_route_manage.xml`、`item_route_config.xml`、`RouteManageActivity.kt`、`RouteConfigAdapter.kt`。
- 影响路线编辑页：`activity_route_edit.xml`、`RouteEditActivity.kt`、路线保存校验与可能的 repository 查询逻辑。
- 不引入新的外部依赖，不改变 Citybus 查询接口和地点搜索接口请求参数。
