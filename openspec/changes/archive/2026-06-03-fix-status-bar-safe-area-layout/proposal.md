## 为什么

移除系统 ActionBar 后，页面根布局仍使用固定 `20dp` padding，没有处理手机状态栏安全区域，导致顶部标题和按钮在部分设备上与状态栏重叠或贴得过近。需要优化页面安全区域和顶部间距，让主界面、路线管理页和路线编辑页在真机/模拟器上都有稳定、舒适的布局。

## 变更内容

- 为主要页面增加状态栏安全区域处理，避免内容与系统状态栏重叠。
- 优化主界面顶部标题、管理按钮、路线选择器和查询按钮的位置。
- 同步优化路线管理页和路线编辑页顶部内容位置，避免不同页面出现同类问题。
- 保持现有功能不变：路线配置、Mock 查询、结果表格和排序逻辑不受影响。
- 构建后需要在模拟器或真机进行视觉验证。

## 能力

### 新增能力

- `safe-area-layout`：定义页面内容需要避开系统状态栏，并保持标题、按钮和主要控件在合适位置展示。

### 修改能力

无。

## 影响范围

- 可能修改主界面、路线管理页和路线编辑页的 XML 根布局或外层容器。
- 可能新增 Kotlin 工具方法，用于统一应用 window insets / status bar padding。
- 可能修改 `MainActivity`、`RouteManageActivity`、`RouteEditActivity` 以应用安全区域。
- 不影响数据模型、SQLite、本地仓库、Mock 查询和排序实现。
