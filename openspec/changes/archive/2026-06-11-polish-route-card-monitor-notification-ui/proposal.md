## Why

目前路線結果卡片右側候車區同時容納主候車文字、下一班摘要和通知鈴鐺，實際文字寬度偏小，容易把固定文案中的 `分鐘` 截斷。通知欄監控目前雖然能展示狀態、候車、步行、下一班和更新時間，但折疊態信息主次不夠清楚；用戶在通知欄或鎖屏上最需要先看到「現在處於哪個出門狀態」，再快速讀到距離最遲出門時間還剩多久或已過多久。

本變更以小範圍 UI polish 修正這兩個問題：讓兩位數候車文案完整展示，並將通知欄監控改為標準通知模板下的「狀態標題 + 倒計時正文」。

## What Changes

- 適度增加路線卡片右側候車資訊塊寬度，支持正常字體與常見手機寬度下完整展示 `等候 NN 分鐘` 和 `下一班 NN 分鐘 ›`。
- 保留通知鈴鐺不小於 48dp 的觸控區，不縮小鈴鐺可點範圍。
- 候車區不需要支持三位數分鐘完整展示；超出兩位數的長文案可按既有降級策略處理。
- 在極窄屏或系統超大字體下，優先完整展示主候車狀態；下一班摘要可隱藏或整行省略，但不得只截斷到半個單位。
- 通知內容標題改為 `<首程巴士號> · <監控狀態>`，例如 `82X · 準備出門`、`82X · 立即出門`、`82X · 快遲到了`。
- 折疊態正文按 `剩餘/已過 → 車 X 分鐘到 → 步行/下一班` 排序，讓用戶先看到距離最遲出門時間的倒計時。
- `準備出門` 和 `立即出門` 折疊態展示 `剩餘 N 分鐘 · 車 X 分鐘到 · 步行 M 分鐘`。
- `快遲到了` 折疊態展示 `已過 N 分鐘 · 車 X 分鐘到 · 下一班 Y 分鐘`，優先給出備選車信息。
- 展開態使用標準 `BigTextStyle`，展示 `剩餘/已過 N 分鐘 · 車 X 分鐘到 · 步行 M 分鐘 · 下一班 Y 分鐘 · HH:mm 更新`。
- 不修改通知渠道名稱、系統頂部可能展示的 App 名稱或 notification channel 名稱。
- 通知 action 只保留 `刷新` 和 `停止`，不再提供 `打開 App` action；點擊通知本體仍可進入 App。
- 暫不做 custom `RemoteViews` 或自定義通知卡片，先使用 Android 標準通知模板。

## Capabilities

### New Capabilities
- `notification-monitor-title`: 定義通知欄監控通知標題、折疊態正文、展開態正文和 action 的信息排序。

### Modified Capabilities
- `bus-route-result-table`: 結果卡片候車資訊塊需要支持兩位數主副候車文案完整展示，並保留鈴鐺觸控區。
- `route-query-results-layout`: 結果卡片布局需要在常見手機寬度下為右側候車資訊塊提供穩定寬度，同時保持左側路線與站點預覽可讀。

## Impact

- 影響 UI layout：`item_bus_route.xml` 中候車資訊塊、左側文字列保留空間和鈴鐺位置約束需要微調。
- 影響 Adapter 或 layout 測試：需覆蓋候車文字完整顯示、鈴鐺觸控區和極端字體降級策略。
- 影響通知 formatter：需要從首程巴士號、監控狀態、第一班 ETA、步行時間、下一班 ETA 和更新時間生成折疊態與展開態文案。
- 影響通知構建：`BusMonitorService` 應使用格式化後的 content title、content text 和 `BigTextStyle` 文案，不再拼接 `通知欄監控` 作為標題。
- 影響通知 action：移除 `打開 App` action，只保留 `刷新` 和 `停止`；通知本體點擊仍保留打開 App 能力。
- 不新增網絡請求、不改變 ETA 刷新策略、語音播報或停止策略。
