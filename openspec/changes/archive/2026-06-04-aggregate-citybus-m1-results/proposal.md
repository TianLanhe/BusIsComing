## Why

当前 Citybus 路线查询只使用单一 `m1=T` 模式，实际返回结果会遗漏其他排序/筛选模式下的可选路线，导致主界面无法展示完整候选路线。用户已确认 `m1` 的 `W`、`F`、`T` 三个枚举值会带来不同路线集合，因此需要并发查询并聚合结果。

同时，Citybus 返回中已经出现“免費”价格和“步行距離(約) XXX米”信息。现有解析和表格模型还不能正确表达这些结果，需要同步扩展。

## What Changes

- 路线查询从单次 HTTP 请求改为并发发起 3 次请求：
  - `m1=T`
  - `m1=F`
  - `m1=W`
- 三次请求共用同一个查询时间 `t`，仍使用当前香港时间格式 `yyyy-MM-dd HH:mm`。
- 其他 query 参数保持当前 App 逻辑不变，不新增 `ws`、`loc`、`ssid`、`sysid`。
- 聚合三次返回的候选路线，并按 `路线号序列 + 总价格 + 总耗时 + 步行距离` 完全一致去重。
- 聚合后的默认结果按总耗时由低到高排序，并让表头状态体现当前排序为 `总耗时(分钟)` 升序。
- 支持解析每条路线的步行距离：
  - 从 `步行距離(約) XXX米` 中提取 `XXX`。
  - 主界面结果表格新增第 5 列 `步行距离(米)`。
  - 第 5 列支持点击表头升序/降序排序。
- 支持解析免费价格：
  - `免費` 或 `免費 *` 按 `0.0` HKD 计入总价。
  - 价格列仍只展示数值，例如 `8.1`；全免费路线显示 `0.0`。
- 结果表格允许横向滚动，以保证 5 列在手机宽度下可读。
- 增加测试路线验证：
  - 起点：`漁灣邨漁進樓`，坐标 `22.267079693838, 114.24208950984`
  - 终点：`港鐵中環站`，坐标 `22.282043425996, 114.15760138031`
  - 验证可展示多程巴士路线、三种 `m1` 返回差异、步行距离和免费价格解析。

## Capabilities

### New Capabilities

- `citybus-m1-result-aggregation`：定义 Citybus 路线查询并发请求 `m1=T/F/W`、聚合去重、默认排序、免费价格解析和步行距离解析规则。
- `bus-route-walking-distance-table`：定义主界面 5 列横向滚动结果表格、步行距离展示和五列排序规则。

### Modified Capabilities

无。

## Impact

- 修改 `CitybusBusRouteRepository` 的查询流程，从单次请求改为三次并发请求、聚合、去重和默认排序。
- 修改 `CitybusRouteParser`，支持 `免費` 价格、步行距离提取，以及新字段输出。
- 修改 `BusRouteOption`、`SortField`、`BusRouteSorter`，增加步行距离字段和排序能力。
- 修改主界面表头、结果行布局和 `BusRouteAdapter`，支持 5 列横向滚动表格。
- 更新 Citybus 请求构造、parser、排序和 UI 相关单元测试。
- 需要在模拟器中使用 `漁灣邨漁進樓 -> 港鐵中環站` 验证真实返回聚合效果。
