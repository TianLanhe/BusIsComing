## 验证记录

- 单元测试：`./gradlew testDebugUnitTest` 通过，覆盖 URL 参数、T/F/W 样例 parser、仓库三模式聚合、去重、部分失败、全失败和空结果。
- 完整构建：`./gradlew build` 通过，覆盖编译、单元测试、lint、debug/release assemble。
- OpenSpec：`openspec validate add-citybus-ws-query-param --strict` 通过。
- 真实 cURL 固定查询时间：`2026-06-04 10:36`，起点 `22.267079693838,114.24208950984`，终点 `22.282043425996,114.15760138031`。
- 不带 `ws` baseline：
  - `m1=T`：HTTP 200，响应 7327 bytes，2 条候选：`788`、`780`。
  - `m1=F`：HTTP 200，响应 7321 bytes，2 条候选：`780`、`788`。
  - `m1=W`：HTTP 200，响应 7324 bytes，2 条候选：`788`、`780`。
- 带 `ws=1.3`：
  - `m1=T`：HTTP 200，响应 7345 bytes，2 条候选：`788`、`780`；当前时间该模式未增加候选。
  - `m1=F`：HTTP 200，响应 12020 bytes，4 条候选：`8X → 10`、`8X → 1`、`788`、`780`，免费转乘段按 0 计入总价。
  - `m1=W`：HTTP 200，响应 30522 bytes，12 条候选，包含 `8 → 90`、`8 → E11`、`8 → 26`、`8 → 25`、`8 → 10`、`8 → 5B`、`8 → 37A`、`8 → 5X`、`8 → 914`、`8 → 930X`、`788`、`780`。
- 真实 HTTP 对照结论：在同一坐标、同一查询时间和同一 `m1` 下，新增 `ws=1.3` 后 `m1=F` 从 2 条增加到 4 条，`m1=W` 从 2 条增加到 12 条；带 `ws=1.3` 的三种 `m1` 返回集合存在差异。
- Pixel 8 模拟器验证：复用测试路线 `中环聚合测试：漁灣邨漁進樓 -> 港鐵中環站`，安装 debug 包后发起真实查询。
- 模拟器查询结果：UI 默认按 `总耗时(分钟) ↑` 排序，展示了 `788`、`780`、`8X → 10`、`8X → 1`、`8 → 37A`、`8 → 5B`、`8 → 914`、`8 → 90` 等多模式聚合结果；`8X → 10` 和 `8X → 1` 价格列显示 `8.1`。
- 模拟器横向滚动：可见 `预计汽车到站时间(分钟)` 和 `步行距离(米)`，步行距离列显示 `350`、`488`、`355`、`450`、`169`、`167`、`210`、`95` 等真实提取数值。
- 模拟器排序：点击 `步行距离(米)` 表头后，表头显示 `步行距离(米) ↑`，可见行按 `95, 95, 164, 164, 167, 167, 169, 210, 210` 升序排列。
- 实现补充：`ws=1.3` 后 W 模式响应体显著变大，为提高模拟器真实查询稳定性，Citybus HTTP connect/read timeout 从 10 秒调整为 20 秒。
