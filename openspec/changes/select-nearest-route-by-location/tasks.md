## 1. 定位依賴與權限基礎

- [ ] 1.1 在 `gradle/libs.versions.toml` 和 `app/build.gradle.kts` 新增 Google Play Services Location 依賴，確認不引入 Kotlin Android 插件衝突。
- [ ] 1.2 在 `app/src/main/AndroidManifest.xml` 宣告 `ACCESS_FINE_LOCATION` 和 `ACCESS_COARSE_LOCATION` 前台定位權限。
- [ ] 1.3 在主頁定位流程中沿用 `ActivityCompat.requestPermissions(...)` 與 `onRequestPermissionsResult(...)` 風格，新增定位權限 request code 並避免影響既有通知權限流程。

## 2. 最近路線選擇策略

- [ ] 2.1 新增純邏輯策略類，輸入目前位置經緯度、水平精度和按常用排序排列的 `RouteConfig` 列表，輸出是否自動選中及選中 route id。
- [ ] 2.2 實作直線距離計算與 tie-breaker：距離相同或無法明確區分時保留傳入列表順序，不修改完整列表排序。
- [ ] 2.3 實作精度規則：`accuracy <= 500m` 直接選最近；`accuracy > 500m` 時僅在最近起點比第二近起點至少近 `accuracy` 米時選中。
- [ ] 2.4 覆蓋無效定位、缺少精度、少於 2 條路線、精度不足和明顯領先等 fallback 結果。

## 3. 主頁定位生命週期與降級

- [ ] 3.1 在 `MainActivity` 中接入 `FusedLocationProviderClient`，以 `PRIORITY_HIGH_ACCURACY` 發起一次目前位置請求，並套用 3 秒超時。
- [ ] 3.2 限制每次 `MainActivity` 建立後最多自動定位一次；從管理頁返回同一 Activity 時不重新定位。
- [ ] 3.3 常用路線少於 2 條時不請求權限、不定位、不顯示 `附近`，保持既有預設選中行為。
- [ ] 3.4 定位進行中先按既有常用排序顯示預設選中路線，不新增 `定位中` UI。
- [ ] 3.5 若定位結果返回前用戶已手動選擇路線，丟棄該次定位結果，不覆蓋用戶選擇。
- [ ] 3.6 Google Play Services Location 不可用、定位失敗、定位超時或定位結果為空時，不回退 `LocationManager`，保持常用排序 fallback。

## 4. 權限拒絕記錄與 Toast 限頻

- [ ] 4.1 使用 `SharedPreferences` 持久化用戶拒絕自動定位權限請求的狀態。
- [ ] 4.2 首次符合條件且無定位權限時請求精確和粗略定位；用戶拒絕後，後續未授權狀態不再自動彈權限請求。
- [ ] 4.3 若用戶後續在系統設定重新授權，下一次主頁建立時恢復自動定位流程。
- [ ] 4.4 將「僅本次允許」視為臨時授權；下次權限過期後仍可再次自動請求，不記為永久拒絕。
- [ ] 4.5 實作三類 fallback Toast 文案及同一 App 進程內每類最多一次的限頻：未允許定位、暫時無法取得目前位置、目前位置不夠精確。

## 5. 主頁快捷卡 UI

- [ ] 5.1 在 `MainActivity` 增加臨時 `nearbySelectedRouteId` 類似狀態，表示本次自動定位選中的路線。
- [ ] 5.2 在 `createRouteShortcutCard(...)` 中僅對目前選中且由定位選中的主頁快捷卡顯示 `附近` 標籤。
- [ ] 5.3 定位成功且最近路線就是既有預設選中路線時，仍顯示 `附近` 標籤。
- [ ] 5.4 用戶從快捷卡或完整列表手動選擇任一路線時，清除 `附近` 標籤。
- [ ] 5.5 保持完整 `全部` 路線列表既有樣式，不在列表項中顯示 `附近` 標籤。
- [ ] 5.6 確認自動選中非原始 Top 3 路線時沿用現有臨時提升規則，不修改 `RouteConfigRepository` 排序和使用統計。

## 6. 單元測試與回歸

- [ ] 6.1 新增或更新策略單元測試，覆蓋少於 2 條路線不定位、精度小於等於 500 米、粗略位置明顯領先、精度不足和無效定位。
- [ ] 6.2 新增或更新 tie-breaker 測試，確認距離相同或接近時按既有常用排序選擇。
- [ ] 6.3 新增或更新主頁選擇狀態測試，確認自動選中不記錄使用統計、不改完整列表排序，且非 Top 3 路線臨時提升。
- [ ] 6.4 新增或更新手動選擇與過期定位結果測試，確認手動選擇後丟棄延遲定位結果並清除 `附近`。
- [ ] 6.5 新增或更新 Toast 限頻與權限拒絕狀態測試，確認三類 fallback 提示各自限頻且拒絕狀態可持久化。
