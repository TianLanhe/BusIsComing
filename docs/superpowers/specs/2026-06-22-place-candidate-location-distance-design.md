# 候選地點 GPS 距離與共用定位設計

日期：2026-06-22
對應 OpenSpec change：`select-nearest-route-by-location`

## 背景

Citybus 地點搜尋結果已提供每個候選的名稱、緯度與經度，但新增、編輯、複製路線及臨時查詢目前只展示地點名稱。香港存在不少同名、相近名稱或分散於不同屋苑位置的候選，用戶難以僅憑名稱判斷哪個候選更接近自己。

專案同時已有「主頁依 GPS 自動選擇最近常用路線」的 OpenSpec change。兩項需求都需要一次性前台定位與直線距離計算，因此合併到 `select-nearest-route-by-location`，共用定位、快取、超時和距離算法，避免建立兩套能力。

## 已確認範圍

- 覆蓋新增、編輯、複製路線及臨時查詢。
- 距離是 GPS 目前位置到 Citybus 候選座標的直線距離，不是步行或道路距離。
- 距離只作參考，不改變 Citybus 候選順序。
- 路線表單和臨時查詢不主動請求定位權限；只有 App 已有粗略或精確前台定位權限時才顯示。
- 無權限、定位失敗或超時時靜默省略距離，不增加提示、按鈕、Toast 或 placeholder。
- 選定候選後，輸入框仍只顯示地點名稱。

## 定位資料流

```text
MainActivity / RouteEditActivity / TemporaryRouteBottomSheet
                         │
                         ▼
             共用一次性定位協調元件
                         │
       ┌─────────────────┼─────────────────┐
       │                 │                 │
   無權限            快照 <= 30 秒       無有效快照
   立即降級            立即返回          新定位 <= 3 秒
       │                 │                 │
       └─────────────────┴─────────────────┘
                         ▼
               共用位置快照模型
       latitude / longitude / accuracy / age
                         │
                         ▼
                共用直線距離計算器
                    ┌────┴────┐
                    ▼         ▼
              主頁最近路線   候選距離
              保留精度規則   任意成功位置均顯示
```

共用定位元件不直接彈權限視窗。主頁最近路線流程可以按既有設計請求權限；候選相關畫面只消費現有權限。最近 30 秒的位置可跨同一 App 進程中的畫面復用；多個同時需求共用同一次底層定位請求。

## 主頁與候選的精度差異

兩個使用場景有意採取不同規則：

- 主頁最近路線會自動改變目前選中路線，因此保留 `accuracy <= 500m` 或粗略位置「最近起點明顯領先」的可信度判斷。
- 候選距離不會自動選擇或排序，只提供參考；只要成功取得位置，粗略或精確位置均顯示距離，不添加「約」字樣。

## 距離格式

距離先四捨五入為整數米，再決定單位：

- 取整後 `< 1000m`：顯示整數米，例如 `368m`。
- 取整後 `>= 1000m`：顯示一位小數公里，例如 `1.0km`、`1.2km`。
- 數字與單位之間不加空格。

邊界示例：

| 原始距離 | 顯示 |
|---|---|
| 368.4m | `368m` |
| 999.4m | `999m` |
| 999.6m | `1.0km` |
| 1249.6m | `1.3km` |

## 候選項 UI

候選項保持約 52dp 單行結構：

```text
杏花邨商場                       [定位圖示] 368m
華富（二）邨華興樓              [定位圖示] 1.2km
```

- 地點名稱：左側 16sp 主文字，佔用剩餘空間，過長時尾端省略。
- 距離：右側約 14dp 綠色線框定位圖示加 13sp 次要文字，完整顯示且不換行。
- 地點名稱前不加圓點或圖示。
- 不增加「距你」「約」等前綴。
- 右側距離不得令候選改成雙行或降低既有 4 至 6 個完整候選的可見目標。
- 定位圖示視為裝飾，候選行的無障礙描述朗讀完整地點名稱及「距離目前位置 X 米／公里」。

## 非阻塞更新

候選搜尋不等待定位：

1. Citybus 候選返回後立即顯示名稱。
2. 位置已可用時同步顯示距離。
3. 位置稍後返回時，只在原有候選行補上距離。
4. 不重新提交或重新排序候選，不關閉列表，不改變滾動位置。
5. 頁面或彈層關閉後忽略遲到位置結果。

## 實作邊界

- 位置快照、定位協調、直線距離計算與格式化保持獨立、可 JVM 單測。
- `PlaceInputController` 只接收可選位置快照並更新候選 presentation，不自行請求權限。
- `Place` 模型、Intent extra、路線配置及 SQLite schema 不增加距離欄位。
- 位置和計算結果不傳送給 Citybus，也不持久化。
- 共用 Haversine 計算器同時供最近路線、候選距離及現有通知監控距離估算使用。

## Change 順序

`improve-route-refresh-feedback` 尚有候選視覺修正任務，且其規格暫時寫明「候選項只顯示地點名稱」。實作順序必須為：

1. 先完成候選正式卡片、52dp 行高、4 至 6 項高度、巢狀滾動及交換按鈕行為。
2. 再 apply `select-nearest-route-by-location`，由本 change 明確取代「只顯示名稱」限制並加入右側距離。

這可避免兩個 change 同時重寫候選 item，亦避免距離功能建立在即將被替換的臨時表格行上。

## 驗證

- JVM 單元測試：30 秒快照、3 秒超時、並發請求共用、Haversine、主頁精度規則及距離格式邊界。
- Instrumentation：候選先顯示、距離原位補充、不重排、不改滾動位置、頁面關閉後忽略結果。
- 權限狀態：無權限、粗略權限、精確權限、定位關閉、超時及空結果。
- 流程：新增、編輯、複製及臨時查詢的搜尋、交換、選擇、保存與返回。
- 視覺：52dp 單行、長名稱省略、距離完整、定位圖示位置、4 至 6 個完整候選、窄屏與大字體。
- 模擬器 GPS：米／公里格式及主頁最近路線選擇。
- 最終運行完整 instrumentation、`./gradlew build` 與 `openspec validate select-nearest-route-by-location --strict`。
