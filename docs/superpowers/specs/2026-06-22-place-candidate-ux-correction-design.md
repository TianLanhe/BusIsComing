# 地點候選輸入體驗驗收修正設計

日期：2026-06-22
對應 OpenSpec change：`improve-route-refresh-feedback`

## 背景

`improve-route-refresh-feedback` 將新增、編輯、複製路線及臨時查詢的地點候選由獨立下拉視窗改為表單內嵌 `RecyclerView`。實機截圖顯示目前實作出現三項驗收問題：

1. 起點和終點輸入框的垂直高度、內邊距與文字層級塌縮，明顯偏離改動前的 Material 輸入框觀感。
2. 候選列表使用臨時表格行樣式，缺少與專案一致的卡片層次、圓角、分隔和按壓反饋。
3. 高度算法以整個可視屏幕為基準，沒有計算候選列表實際位置到輸入法頂部的剩餘空間；因此程式認為至少容納三項，畫面實際只露出兩項。

此外，路線編輯頁的外層仍是一般 `ScrollView`，而內部候選是可獨立滾動的 `RecyclerView`。這不是上述三項問題的直接根因，但會帶來手勢分發、快速滑動和輸入法尺寸變化時的巢狀滾動風險。

## 目標

- 恢復改動前清晰、舒適的輸入框高度、浮動標籤、內邊距和文字基線。
- 保留內嵌候選列表，避免重新引入會與輸入法重疊的 PopupWindow。
- 將候選列表調整為專案既有「安靜實用的現代通勤工具」風格。
- 使用候選實際位置與輸入法頂部計算可用高度；主驗收配置顯示四至六項，替代驗收配置至少顯示三項。
- 讓候選列表和外層表單的縱向滑動可靠協作。
- 統一新增、編輯、複製路線及臨時查詢的視覺和互動。

## 非目標

- 不恢復 `MaterialAutoCompleteTextView.showDropDown()`。
- 不引入 Compose、自訂複合 View 或第三方候選元件。
- 不改動地點搜尋 API、debounce、generation、最多 100 條資料、校驗、交換、保存或查詢協議。
- 不重新設計整個路線編輯頁或臨時查詢底部彈層。
- 不為候選項增加地點圖標、圓點、副標題或其他目前資料模型沒有的資訊。

## 已確認設計

### 1. 輸入框視覺

保留普通 `OutlinedBox` 語義，不再依賴 `ExposedDropdownMenu` 樣式，也不顯示下拉箭頭。建立新增／編輯／複製路線與臨時查詢共用的地點輸入 style，明確恢復：

- 56dp 最小觸控高度。
- 內邊距與文字垂直位置。
- 浮動標籤與邊框層級。
- 16sp 輸入文字、單行顯示及尾端省略。

輸入框橫向寬度和既有右側 56dp 交換按鈕區保持不變。

### 2. 候選列表位置

候選列表緊跟目前聚焦的輸入框下方。當起點候選展開時，終點輸入框自然向下移；當終點候選展開時，候選位於終點下方。同一時間只顯示一個候選列表。

候選顯示期間暫時隱藏交換按鈕，避免按鈕懸在輸入框與候選之間或隨候選高度跳動。候選關閉或選定後，交換按鈕恢復到兩個輸入框的中線位置。

### 3. 候選列表視覺

候選列表使用與輸入框同寬的白色卡片：

- 8dp 圓角。
- 1dp 淺色描邊。
- 2dp elevation 的輕微陰影。
- 候選項高度約 52dp。
- 地點名稱使用 16sp、單行、省略號截斷。
- 項目之間使用淺色分隔線。
- 提供清晰但克制的按壓／點選狀態。
- 候選項只顯示地點名稱，不顯示圓點、圖標或額外描述。

loading、無結果和失敗繼續使用目前輸入區域的 helper/error 文案與位置，不新增狀態卡。

### 4. 高度和外層滾動

候選高度不再使用「整個可視屏幕高度的 40%」。展示候選時：

1. 取得輸入法頂部位置。
2. 必要時以最小幅度向上滾動外層表單，使目前輸入框和候選有足夠空間。
3. 以「輸入法頂部 − 候選列表頂部 − 8dp 安全間距」計算真實可用高度。
4. 將可用高度換算為完整候選行數。
5. 在 1080 × 2400、字體縮放 1.0 的主驗收配置顯示四至六項；在 1080 × 1920、字體縮放 1.15 的替代驗收配置至少顯示三項；任何配置最多顯示六項，資料仍最多保留 100 條。

候選超出高度後由內部 `RecyclerView` 滾動。候選關閉後不強制恢復外層表單原滾動位置，以避免畫面來回跳動。

### 5. 巢狀滾動

`activity_route_edit.xml` 根容器由一般 `ScrollView` 換為 AndroidX `NestedScrollView`，保留原 ID、`fillViewport`、背景和內容結構。

候選 `RecyclerView` 保持巢狀滾動能力：

- 候選仍可滾動時，手勢優先滾動候選。
- 候選到達頂部或底部後，剩餘手勢可自然傳遞給外層表單。
- 輸入法尺寸改變或外層最小幅度移動時，不應出現明顯跳動。

臨時查詢繼續使用 `NestedScrollView` 和接近全螢幕的候選模式。

### 6. 焦點與狀態

- 選擇一端後，若另一端尚未選定，保留輸入法並自動聚焦另一端。
- 另一端已有文字，且 controller 目前保存的候選仍對應該文字與最新 search generation 時，聚焦後立即重新顯示候選；若為空或候選已失效，只聚焦並等待輸入。
- 兩端均已選定時，不自動提交、不切換焦點、不收起輸入法。
- 點擊空白處或第一次返回只關閉候選並保留文字。
- 候選關閉後，返回才沿用 Activity 或 BottomSheet 原有行為。
- debounce、過期結果忽略、loading／無結果／失敗文案、選擇失效、交換、校驗、保存和查詢語義保持不變。

## 方案比較

### 採用：普通 OutlinedBox + 明確尺寸和樣式

視覺可恢復至舊版，同時保持「可輸入文字 + 內嵌候選」的正確元件語義；不依賴 Material 下拉菜單內部預設，升級風險較低。

### 未採用：恢復 ExposedDropdownMenu 樣式並隱藏箭頭

改動較少，但仍借用下拉菜單樣式承載非下拉菜單行為，後續 Material 版本變更可能再次造成尺寸或 padding 偏差。

### 未採用：建立全新自訂複合 View

長期封裝更完整，但超出本次驗收修正範圍，增加遷移和回歸成本。

## 測試與驗收

- 單元測試覆蓋最多 100 條、真實剩餘高度換算、三至六條完整行及邊界值。
- UI 測試驗證輸入框實際高度、橫向寬度和浮動標籤，不只比對 XML 字串。
- UI 測試驗證候選卡樣式、約 52dp 行高、單行省略、沒有圖標／圓點及可點選狀態。
- 模擬器分別驗證新增、編輯、複製和臨時查詢。
- 驗證起點與終點候選都緊跟目前輸入框，另一輸入框被自然下推。
- 在 1080 × 2400、字體縮放 1.0 驗證四至六條完整候選；在 1080 × 1920、字體縮放 1.15 驗證至少三條。
- 驗證候選內部滾動、到邊界後外層 `NestedScrollView` 接管手勢。
- 驗證頁面只做必要的最小幅度上移，候選關閉後不強制跳回。
- 驗證交換按鈕在候選期間隱藏，候選關閉後恢復。
- 驗證焦點切換時已有有效候選立即恢復，空輸入不顯示候選。
- 保留常規尺寸與較小屏幕／較大字體截圖。
- 完成後運行完整 instrumentation、`./gradlew build` 和 `openspec validate improve-route-refresh-feedback --strict`。
